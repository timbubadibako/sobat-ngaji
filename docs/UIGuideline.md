# UI Guideline

## UI References
Referensi HTML prototype berada di folder `design/`. Implementasi Flutter harus mengikuti style yang sudah disetujui dari component showcase dan design guideline.

## Main Navigation
Bottom navigation final:
- Home
- Practice
- Insight
- Profile

## Home UI
- Daily Qira sebagai centerpiece.
- AI presence muncul lewat greeting, recommendation, dan progress.
- Jangan membuat Home seperti dashboard statistik.

## Practice UI
- Fokus pada listen, record, evaluate.
- Audio reference harus dekat dengan Arabic text.
- Recording control harus mudah dijangkau ibu jari.

## Insight UI
- Letter mastery harus mudah dipahami.
- Chart tidak boleh dekoratif tanpa insight.
- Smart suggestion harus menjawab next practice.

## Profile UI
- Ringkas.
- Tidak menjadi halaman setting berat.
- Learning powered by AI boleh muncul sebagai identity cue.

## Component Consistency
- Button style tidak boleh berbeda antar feature.
- Card radius default 18.
- Hero radius 24.
- Chip menggunakan pill radius.
- Badge semantic mengikuti DesignGuideline.
